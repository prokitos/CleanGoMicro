cmd = точка входа
pkg = файлы, библиотеки, код, который используется в других проектах.
api = документация или сваггеры.
config = yaml или env файлы.

internal = внутренняя логика приложения.
  app = здесь собираются логика и зависимости, и запускают приложение.
  models = структуры ответов, таблиц и тому подобное.
  config = инициализация конфигов из yaml файлов.
  services = бизнес логика приложения. (уровень бизнес логики)
  transport = роуты, сервер, rmq	(транспортный уровень)
  database  = методы для взаимодействия с бд (уровень базы данных)


internal 
   транспортный уровень.  пользователь взаимодействует с приложением.  
   уровень бизнес логики. основная логика приложения.
   уровень базы данных.   чтения или запись в базы данных.






При запуске микросервиса, запускается файл cmd/main.go
  он загружает конфиги internal/config/config.go из config/local.yaml
  он запускает миграцию и открывает соединения с бд из internal/database/
  он запускает сервер из internal/app/run.go

При получении запроса, они попадают в роуты внутри папки internal/transport
  в transport происходит создание таблицы, и её передача в internal/services
    в services поисходит вызов методов у таблицы, и передачей туда ДАО и подключения. Методы таблицы в internal/models/tables
      в методах таблицы происходит вызов crud из переданного ДАО.
        в методах ДАО происходит работа с базой данных, и возвращается ответ пользователю.
